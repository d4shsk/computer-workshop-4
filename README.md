#### 2\. Описание реализации API

В приложении реализовано два ключевых серверных маршрута:

1.  **GET `/login`**:

      * Возвращает JSON-объект с идентификатором автора в системе MOODLE.
      * Формат ответа: `{"author": "1160468"}`.
      * Заголовки ответов настроены на `application/json`.

[![image.png](https://i.postimg.cc/PJkRhz1Y/image.png)](https://postimg.cc/hzpr1myt)

2.  **POST `/size2json`**:

      * Принимает изображение через форму в формате `multipart/form-data` (поле `image`).
      * Валидация: На стороне сервера реализована проверка типа файла. Если передан не PNG или файл поврежден, возвращается JSON `{"result": "invalid filetype"}`.
      * Обработка: С помощью библиотеки `image-size` сервер считывает буфер изображения и извлекает его реальную ширину и высоту.
      * Лимиты: Установлено ограничение на размер файла (10 МБ). При превышении возвращается сообщение `File too large`.

#### 3\. Реализованные усложнения

Проект выполнен с максимальным уровнем сложности:

  * Реализация фронтенда: Разработан пользовательский интерфейс для взаимодействия с сервером без использования сторонних утилит (типа Postman).
  * Асинхронная отправка данных: Загрузка изображения и получение результата происходят через `Fetch API`. Страница не перезагружается при отправке формы.
  * Сохранение состояния: Использовано `localStorage` для кэширования последнего результата. При обновлении страницы приложение «помнит» габариты последнего успешно обработанного файла.
  * Отображение Thumbnail: Реализовано создание временной ссылки на файл через `URL.createObjectURL`, что позволяет пользователю видеть превью изображения сразу после выбора файла.

#### 4\. Команды управления проектом

  * `npm install` — установка зависимостей.
  * `node server.js` — запуск сервера.
  * Деплой: Приложение развернуто на платформе Render с автоматической поддержкой HTTPS-протокола.
